┌──────────────────────────────────────────────────────────────────────────────┐
│ PHASE 1: PREPARATION (5 minutes)                                             │
└──────────────────────────────────────────────────────────────────────────────┘

GITHUB ACCOUNT SETUP
  □ Create GitHub account: https://github.com/signup
  □ Verify email address
  □ Create new repository: DisasterSAS
  
GIT CONFIGURATION
  □ Configure git locally:
    git config --global user.name "Your Name"
    git config --global user.email "your@email.com"

GENERATE PRODUCTION SECRETS
  □ Run: bash generate-secrets.sh
  □ Copy output to notepad/document
  □ Keep secrets safe!

┌──────────────────────────────────────────────────────────────────────────────┐
│ PHASE 2: GITHUB SETUP (10 minutes)                                           │
└──────────────────────────────────────────────────────────────────────────────┘

PUSH CODE TO GITHUB
  □ Navigate to your project directory
  □ Run these commands:
    
    git init
    git add .
    git commit -m "Initial commit: Disaster Safety App"
    git branch -M main
    git remote add origin https://github.com/YOUR_USERNAME/DisasterSAS.git
    git push -u origin main

VERIFY GITHUB REPOSITORY
  □ Visit: https://github.com/YOUR_USERNAME/DisasterSAS
  □ Confirm files uploaded:
    ✓ Dockerfile
    ✓ docker-compose.yml
    ✓ nginx.conf
    ✓ disaster_backend/ (with app.py, requirements.txt)
    ✓ disaster_frontend/ (HTML files)
    ✓ .env.example (but NOT .env)
    ✓ .dockerignore
  
  □ Verify .env is NOT committed (check file list)

TEST LOCALLY BEFORE DEPLOYING
  □ docker compose up -d
  □ docker compose ps (all services Running)
  □ curl http://localhost:5000/api/health (200 OK)
  □ docker compose down

┌──────────────────────────────────────────────────────────────────────────────┐
│ PHASE 3: RAILWAY.APP SETUP (15 minutes)                                     │
└──────────────────────────────────────────────────────────────────────────────┘

CREATE RAILWAY ACCOUNT
  □ Go to: https://railway.app
  □ Click "Sign Up"
  □ Sign in with GitHub
  □ Authorize Railway access

CREATE NEW PROJECT
  □ Click "New Project"
  □ Select "Deploy from GitHub repo"
  □ Search for "DisasterSAS"
  □ Click to connect

ADD BACKEND SERVICE
  □ Click "Add Service"
  □ Select "GitHub Repo"
  □ Choose DisasterSAS repository
  □ Railway auto-detects Dockerfile
  □ Service deploys automatically

CONFIGURE BACKEND ENVIRONMENT VARIABLES
  □ Click on backend service
  □ Go to "Variables" tab
  □ Add these variables (from generate-secrets.sh output):
    
    FLASK_ENV=production
    SECRET_KEY=<your-secret-key>
    JWT_SECRET_KEY=<your-jwt-key>
    BACKEND_PORT=5000
    POSTGRES_DB=disastersas
    POSTGRES_USER=disastersas
    POSTGRES_PASSWORD=<your-password>
    UPLOAD_FOLDER=/app/uploads
    MAX_UPLOAD_BYTES=5242880

SET BACKEND PORT
  □ Click backend service
  □ Go to "Settings"
  □ Set "Port" to: 5000

ADD POSTGRESQL DATABASE
  □ Click "Add Service"
  □ Select "Database"
  □ Choose "PostgreSQL"
  □ Railway creates database automatically

GET DATABASE CONNECTION INFO
  □ Click on PostgreSQL service
  □ Go to "Connect" tab
  □ Copy DATABASE_URL

UPDATE BACKEND DATABASE URI
  □ Click on backend service
  □ Go to "Variables"
  □ Update DATABASE_URI with Railway's DATABASE_URL
  
  Format: postgresql+psycopg2://user:password@host:port/dbname
  
  Or let Railway inject it automatically

VERIFY SERVICES CONNECTED
  □ Backend can see PostgreSQL environment variable
  □ Both services show "Running" status in dashboard

┌──────────────────────────────────────────────────────────────────────────────┐
│ PHASE 4: DEPLOYMENT (5 minutes)                                              │
└──────────────────────────────────────────────────────────────────────────────┘

TRIGGER DEPLOYMENT
  □ Click "Deploy" button on backend service
  □ Watch build progress in logs
  □ Build completes successfully
  □ Services show "Running"

MONITOR BUILD LOGS
  □ Go to "Deployments" tab
  □ View build logs in real-time
  □ Look for errors:
    ✓ "Build successful"
    ✓ "Container started"
    ✓ "[OK] Database Created Successfully"

GET PUBLIC URL
  □ Click backend service
  □ Copy public URL (e.g., https://yourapp-prod.up.railway.app)
  □ Save this URL for testing

┌──────────────────────────────────────────────────────────────────────────────┐
│ PHASE 5: VERIFICATION (5 minutes)                                            │
└──────────────────────────────────────────────────────────────────────────────┘

TEST BACKEND API
  □ Visit: https://yourapp-production.up.railway.app/api/health
  □ Should return:
    {"status": "ok", "message": "Disaster Safety Backend Running Successfully 🚀"}
  
  □ Status code: 200

TEST FRONTEND
  □ Visit: https://yourapp-production.up.railway.app/
  □ Should load index page
  □ No errors in browser console (F12)

CHECK LOGS FOR ERRORS
  □ Click backend service
  □ Go to "Logs" tab
  □ Look for:
    ✓ No "ERROR" messages
    ✓ "Database Created Successfully"
    ✓ Flask running message

VERIFY DATABASE CONNECTION
  □ Backend logs show database connection success
  □ No "connection refused" errors
  □ PostgreSQL service shows "Healthy"

┌──────────────────────────────────────────────────────────────────────────────┐
│ PHASE 6: POST-DEPLOYMENT (Optional)                                          │
└──────────────────────────────────────────────────────────────────────────────┘

SETUP CUSTOM DOMAIN (Optional)
  □ Prepare domain name (disastersas.com, etc.)
  □ In Railway, go to project settings
  □ Click "Add Domain"
  □ Enter your domain
  □ Update DNS A record to Railway's domain
  □ Wait for DNS propagation (5-30 minutes)

SETUP AUTO-DEPLOY
  □ Go to backend service → Settings
  □ Enable "Auto Deploy" on main branch
  □ Future git pushes auto-deploy

SETUP MONITORING (Optional)
  □ Go to project settings
  □ Enable notifications for build failures
  □ Add team members (Settings → Members)

BACKUP DATABASE (Optional)
  □ In PostgreSQL service → Connect tab
  □ Download backup manually
  □ Or setup automatic backups

┌──────────────────────────────────────────────────────────────────────────────┐
│ TROUBLESHOOTING                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

BUILD FAILS
  □ Check "Build Logs" for error message
  □ Common issues:
    - Missing requirements.txt
    - Incorrect Dockerfile syntax
    - Missing environment variables
  □ Fix and push to GitHub
  □ Railway rebuilds automatically

DATABASE CONNECTION ERROR
  □ Verify DATABASE_URI environment variable
  □ Check PostgreSQL is "Running"
  □ Ensure credentials match (.env variables)
  □ View backend logs for error details

PORT ISSUES
  □ Set BACKEND_PORT=5000 in variables
  □ Ensure port 5000 configured in Railway settings
  □ Check Flask is listening on correct port

SLOW DEPLOYMENT
  □ First deployment takes 5-10 minutes
  □ Subsequent deployments are faster
  □ Watch "Deployments" tab for progress

DEPLOYMENT STUCK
  □ Click "Redeploy" button
  □ Or make a new git commit and push
  □ Railway will retry

┌──────────────────────────────────────────────────────────────────────────────┐
│ SUCCESS INDICATORS                                                           │
└──────────────────────────────────────────────────────────────────────────────┘

Your deployment is successful when:
  ✓ Backend service shows "Running"
  ✓ PostgreSQL service shows "Healthy"
  ✓ /api/health returns 200 OK
  ✓ Frontend loads without errors
  ✓ No "ERROR" in backend logs
  ✓ Database created message appears
  ✓ Public URL is accessible
