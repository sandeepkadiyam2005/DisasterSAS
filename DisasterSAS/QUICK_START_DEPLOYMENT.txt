┌──────────────────────────────────────────────────────────────────────────────┐
│ OPTION 1: Railway.app (Easiest - 5 minutes)                                  │
└──────────────────────────────────────────────────────────────────────────────┘

1. Go to https://railway.app
2. Create account (sign in with GitHub)
3. Create new project → Connect GitHub repo
4. Add services:
   ✓ Backend: Dockerfile + Environment variables
   ✓ PostgreSQL: Plugin
5. Deploy → Auto-deploy on git push
6. Done! Your app is live

Cost: $0 (free tier) or $5-20/month

┌──────────────────────────────────────────────────────────────────────────────┐
│ OPTION 2: DigitalOcean (Best Control - 30 minutes)                           │
└──────────────────────────────────────────────────────────────────────────────┘

1. Create account at https://digitalocean.com
2. Create Ubuntu 22.04 droplet ($4/month)
3. SSH into droplet and run:
   
   bash deploy-digitalocean.sh

4. Clone your GitHub repo:
   
   cd /opt/disaster-app
   git clone <your-repo>
   
5. Setup environment:
   
   cp .env.example .env
   nano .env  # Update production secrets

6. Start services:
   
   docker compose up -d

7. Setup domain (optional):
   
   nano /etc/caddy/Caddyfile
   systemctl restart caddy

Cost: $19/month (droplet $4 + managed DB $15)

┌──────────────────────────────────────────────────────────────────────────────┐
│ OPTION 3: Fly.io (CLI-based - 15 minutes)                                    │
└──────────────────────────────────────────────────────────────────────────────┘

1. Install Fly CLI:
   curl -L https://fly.io/install.sh | sh

2. Login:
   flyctl auth login

3. Launch:
   flyctl launch

4. Deploy:
   flyctl deploy

Cost: Free tier or $5+/month

┌──────────────────────────────────────────────────────────────────────────────┐
│ OPTION 4: Render.com (Simple - 10 minutes)                                   │
└──────────────────────────────────────────────────────────────────────────────┘

1. Go to https://render.com
2. Create account (GitHub sign-in)
3. Create services:
   ✓ Web Service (Dockerfile)
   ✓ PostgreSQL
4. Configure environment & deploy
5. Done!

Cost: $7+/month

╔══════════════════════════════════════════════════════════════════════════════╗
║                            BEFORE ANY DEPLOYMENT                             ║
╚══════════════════════════════════════════════════════════════════════════════╝

□ Update .env with PRODUCTION secrets:
  - SECRET_KEY (use: openssl rand -hex 32)
  - JWT_SECRET_KEY (use: openssl rand -hex 64)
  - POSTGRES_PASSWORD (strong password)

□ Push code to GitHub

□ Test locally:
  docker compose up -d
  docker compose logs -f

□ Verify health:
  curl http://localhost:5000/api/health

╔══════════════════════════════════════════════════════════════════════════════╗
║                         DEPLOYMENT CHECKLIST                                 ║
╚══════════════════════════════════════════════════════════════════════════════╝

During Deployment:
□ Set environment variables on platform
□ Configure database connection
□ Setup custom domain (if applicable)
□ Enable HTTPS/SSL (auto on Railway/Render, Caddy on DigitalOcean)
□ Test all endpoints after deployment

After Deployment:
□ Verify services are running: docker compose ps
□ Check logs for errors: docker compose logs -f
□ Test health endpoint: curl /api/health
□ Setup monitoring: ./monitor.sh
□ Setup backups: ./backup.sh
□ Configure CI/CD auto-deploy

║                        HELPFUL COMMANDS                                       ║

Local Development:
  docker compose up -d              # Start services
  docker compose logs -f            # View logs
  docker compose ps                 # Check status
  docker compose down               # Stop services

Deployment (DigitalOcean):
  bash deploy-digitalocean.sh       # Auto-setup VPS
  docker compose up -d              # Start on VPS
  docker compose logs backend       # View backend logs
  ./monitor.sh                      # Health check
  ./backup.sh                       # Backup data

Backup & Recovery:
  ./backup.sh                       # Create backup
  docker compose exec db psql -U disastersas -d disastersas < backup.sql

Generated Strong Secrets:
  openssl rand -hex 32              # SECRET_KEY
  openssl rand -hex 64              # JWT_SECRET_KEY
  openssl rand -hex 16              # POSTGRES_PASSWORD
